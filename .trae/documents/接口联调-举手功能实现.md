# 接口联调计划 - 举手功能实现

## 任务概述
实现课堂页面中辅讲教室举手功能,包括SSE消息处理、举手状态更新和点击"同意互动"时的三个接口调用。

## 实施步骤

### 步骤1: 在 InteractionPanel.vue 中添加数据字段
在 `data()` 中添加以下字段:
- `subscriberInPics`: 数组,用于保存所有已同意互动的辅讲教室信息
- 初始状态为空数组

### 步骤2: 实现 SSE 'participants' 消息处理
在 `InteractionPanel.vue` 的 `mounted()` 生命周期中,完善 `addEventListener('participants')` 的处理逻辑:

1. 解析消息体,遍历参与者列表
2. 对于每个参与者,检查 `pinfoMap.HAND` 字段是否为 "1"
3. 如果是举手状态且 `TEL` 字段与某个辅讲教室的 `phone` 字段匹配,则:
   - 更新该辅讲教室的 `isRaisingHand` 为 `true`
   - 更新该辅讲教室的 `pid` 为消息体中的 `pid`
4. 如果 `HAND` 不为 "1" 或匹配的辅讲教室之前是举手状态,则更新为 `false`

### 步骤3: 实现 handleApproveInteraction 方法
修改 `InteractionPanel.vue` 中的 `handleApproveInteraction` 方法,实现以下逻辑:

1. 根据参数 `classroomId` 找到对应的教室对象
2. 检查教室是否在线,如果离线则提示错误并返回
3. 检查教室是否有 `pid`,如果没有则提示错误并返回
4. 调用三个接口(按顺序):

#### 3.1 调用 setHandRaise 接口
- 接口路径: `/api/teacher/meeting/control/${scheduleId}/hands/${participantId}`
- 请求方法: PUT
- 请求参数:
  - `participantId`: 教室的 `pid` 字段
  - `handsState`: 0 (放下手)

#### 3.2 调用 setCustomPicture 接口
- 接口路径: `/api/teacher/meeting/control/${scheduleId}/custom-picture`
- 请求方法: PUT
- 请求参数构建逻辑:
  - `manualSet`: 1 (手动设置)
  - `multiPicSaveOnly`: false (立即生效)
  - `imageType`: 根据 `subscriberInPics` 数组长度设置英文单词
    - 长度为 1: "One"
    - 长度为 2: "Two"
    - 长度为 3: "Three"
    - 长度为 4: "Four"
    - 长度为 5: "Five"
    - 长度为 6: "Six"
    - 长度为 7: "Seven"
    - 长度为 8: "Eight"
    - 长度为 9: "Nine"
  - `subscriberInPics`: 数组构建规则:
    - 第一个元素:主讲教室信息
      - `index`: 1
      - `subscriber`: 数组形式,包含 `this.mainClassroomPhone`
      - `isAssistStream`: 0
    - 后续元素:遍历 `this.subscriberInPics` 数组,添加已同意互动的辅讲教室
      - `index`: 当前索引 + 1
      - `subscriber`: 数组形式,包含辅讲教室的 `phone`
      - `isAssistStream`: 0
    - 最后一个元素:当前正在同意的辅讲教室
      - `index`: 当前数组总长度
      - `subscriber`: 数组形式,包含当前辅讲教室的 `phone`
      - `isAssistStream`: 0

#### 3.3 调用 muteParticipant 接口
- 接口路径: `/api/teacher/meeting/control/${scheduleId}/mute/${participantId}`
- 请求方法: PUT
- 请求参数:
  - `participantId`: 与 setHandRaise 相同(教室的 `pid`)
  - `isMute`: 0 (解除静音/开麦)

### 步骤4: 更新本地状态
成功调用三个接口后:
1. 将当前辅讲教室添加到 `subscriberInPics` 数组中,保存其 `phone` 字段
2. 更新教室的 `isInteracting` 为 `true`
3. 更新教室的 `isRaisingHand` 为 `false`
4. 显示成功提示信息

### 步骤5: 错误处理
在接口调用过程中:
- 如果任一接口调用失败,显示错误提示
- 不更新本地状态
- 记录错误日志以便调试

### 步骤6: 获取 imageType 的英文对应
创建辅助方法 `getImageType(count)`:
- 输入: 数字(1-9)
- 输出: 对应的英文单词
- 映射关系:
  - 1 → "One"
  - 2 → "Two"
  - 3 → "Three"
  - 4 → "Four"
  - 5 → "Five"
  - 6 → "Six"
  - 7 → "Seven"
  - 8 → "Eight"
  - 9 → "Nine"

## 接口文档参考

### 1. setHandRaise 接口
- 接口文档位置: OpenAPI.json 第245行
- operationId: `setHandsState`
- 请求DTO: `HandsStateReqDTO`
  - `handsState`: integer (举手状态: 0放下手, 1举手)

### 2. setCustomPicture 接口
- 接口文档位置: OpenAPI.json 第356行
- operationId: `setCustomPicture`
- 请求DTO: `CustomMultiPictureReqDTO`
  - `manualSet`: integer (是否手动设置: 1手动设置, 0自动设置)
  - `imageType`: string (画面类型: One, Two, Three, Four, Five, Six, Seven, Eight, Nine)
  - `multiPicSaveOnly`: boolean (是否仅保存画面配置: true仅保存, false立即生效)
  - `subscriberInPics`: array (各画面位置的与会人信息)
    - 每项类型: `SubscriberInPic`
      - `index`: integer (画面位置索引,从1开始)
      - `subscriber`: array (与会人账号列表/SIP号码)
      - `isAssistStream`: integer (是否为辅流: 1是辅流, 0是主流)

### 3. muteParticipant 接口
- 接口文档位置: OpenAPI.json 第34行
- operationId: `muteParticipant`
- 请求DTO: `MuteParticipantReqDTO`
  - `isMute`: integer (是否静音: 1静音, 0解除)

## 数据流说明

1. SSE 消息到达 → 解析 HAND 字段 → 更新教室举手状态
2. 用户点击"同意互动" → 获取教室信息 → 调用三个接口
3. 构建 subscriberInPics 数组 → 包含主讲教室 + 历史互动教室 + 当前教室
4. 根据数组长度确定 imageType → 调用 setCustomPicture
5. 成功后更新本地状态 → 教室变为互动状态

## 注意事项

1. 主讲教室信息必须始终作为 subscriberInPics 的第一个元素
2. index 值从 1 开始递增
3. subscriber 字段是数组格式,需要包含 phone 字符串
4. 每次成功同意互动后,需要将辅讲教室的 phone 添加到 subscriberInPics 数组
5. 需要处理接口调用失败的情况,避免状态不一致
6. SSE 消息处理需要考虑多个参与者的情况,遍历处理每一个
7. 在调用接口前需要验证教室的在线状态和 pid 是否存在
